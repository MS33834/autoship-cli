# 既知の問題

> 本ページでは AutoShip の現在判明している制限事項と未解決問題を正直に記録し、利用への影響を判断しやすくします。「修正予定」と記載した項目は今後のバージョンで対応予定です。

## `verify --fix` は AI バックエンドがないと利用不可

`verify --fix` は AI モデルによる修正提案に依存します。モデルバックエンド（ローカル Ollama クラウド provider いずれも）が未設定の場合、サブコマンドはエラーで終了し、テンプレート修正には**自動ではフォールバックしません**。

- 回避策：`autoship verify pytest`（`--fix` なし）で検証のみ行う；
- または [クイックスタート](quickstart.md) の「+5 分で試す AI あり版」に従い Ollama を設定。
- 状態：仕様であり、バグではありません。

## Windows のパスエッジケース

Windows では以下のパス問題が知られています：

- スペースや非 ASCII 文字を含むユーザーディレクトリ（例：`C:\Users\Zhang San\`）で監査ログのパス解析が異常になることがある；
- `upload --target docker` は PowerShell 7 未満で `\\wsl$\` パスを完全サポートしない；
- `.autoship.toml` のバックスラッシュパスはエスケープするかスラッシュに変更する必要がある。

状態：パス処理の統一は 1.1 で予定。

## 大規模リポジトリでの `clean` パフォーマンス

ファイル数が概ね 5 万超の大規模リポジトリでは、`clean` のフルスキャンに数十秒かかり、メモリ使用量が高くなる場合があります。

- 回避策：`.autoship.toml` の `clean.exclude` で `node_modules`、`venv`、ビルド成果物などをスキップ；
- または `clean --paths src/` でスキャン範囲を限定。
- 状態：増分スキャンと並列化はロードマップ上。

## 翻訳が遅れる場合がある

中国語がソース言語で、英語・日本語は翻訳版です。新機能や改訂は先に中国語ドキュへ反映され、en/ja は数時間〜数日遅れることがあります。

- 影響：リリース直後の短い期間、en/ja ドキュが最新のコマンドやオプションを反映していない場合があります。
- 状態：CI に i18n 完全性チェックを追加し翻訳漏れを警告。mike バージョン管理ドキュ有効化後はさらに厳格化予定。

## mike バージョン管理ドキュは 1.0 以降に有効化

`mkdocs.yml` に `version.provider: mike` を宣言済みですが、バージョンスイッチャーはまだ正式有効化されていません。現在すべての訪問者は最新ビルドを閲覧し、過去バージョンには切り替えられません。

- 状態：1.0 正式リリース後に mike サブドメインとバージョンセレクターを有効化予定；
- それまでは履歴ドキュを閲覧する場合、GitHub の該当 tag 配下の `docs/` ディレクトリを直接参照してください。

## フィードバックと追跡

上記以外の新たな不具合に遭遇した場合は：

- [トラブルシューティング](troubleshooting.md) を確認；
- [GitHub Issues](https://github.com/MS33834/autoship-cli/issues) で検索または issue を作成し、`autoship doctor` の出力と `--verbose` ログを添えてください。
